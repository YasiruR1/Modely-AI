Project Scope

Literature Survey

Automating Business Process Model and Notation (BPMN) diagram generation is becoming essential for businesses to streamline workflow modeling. Traditional manual approaches are time-consuming and prone to errors, resulting in inconsistencies that hinder real-time adaptability and process optimization. Addressing this issue requires innovative technological interventions.

Existing research identifies challenges in converting unstructured text into BPMN models, particularly with ambiguous phrasing and anaphora resolution. Most current methods focus on structured inputs, leaving a gap in handling complex, unstructured business process descriptions. This project aims to bridge that gap using fine-tuned GPT models to automate BPMN generation from text inputs.

By leveraging Natural Language Processing (NLP) and machine learning, this project demonstrates improved accuracy and speed in generating BPMN diagrams. The model's evaluation shows promising results, achieving a precision, recall, and F1 score around 70%. This highlights the potential for AI-driven systems to transform traditional process modeling.

Future work can focus on enhancing the model to handle more complex scenarios, such as real-time data integration and process optimization. This research opens the door for more agile and responsive BPMN generation systems, offering significant improvements to business process automation.

Research Problem & Solution

Proposed Problem

How to automate the generation of BPMN diagrams from unstructured business process descriptions, addressing challenges like ambiguity, anaphora resolution, and the manual effort required in traditional approaches?

Businesses face inefficiencies when creating BPMN diagrams manually due to ambiguous and unstructured text descriptions of their processes. Existing tools struggle with handling complex language elements, resulting in inaccurate or incomplete models. This increases the time and cost of process documentation and reduces responsiveness to real-time changes, highlighting the need for an automated solution.


Product Demonstration - Solution

Proposed Solution

The proposed system leverages Natural Language Processing (NLP) and Large Language Models (LLMs) to automate the generation of Business Process Model and Notation (BPMN) diagrams from unstructured text descriptions. Traditional BPMN diagram creation is time-consuming, prone to errors, and requires specialized expertise. By automating this process, the proposed solution aims to reduce human error and improve scalability, making BPMN modeling more accessible and efficient for businesses. The system first processes text input through advanced NLP techniques to extract key process activities and dependencies. Using a combination of machine learning and semantic analysis, it maps these activities to BPMN elements such as events, tasks, and gateways. A key feature is the detection and elimination of redundant tasks, ensuring that the generated BPMN diagrams are optimized and free from unnecessary processes. The solution integrates a user-friendly interface that allows users to input textual descriptions, visualize the generated diagrams, and make real-time modifications. This flexibility ensures that users can adjust and optimize their processes during the drafting phase, leading to more accurate and relevant BPMN diagrams. The system also supports advanced features like process updates and customization for industry-specific requirements. In conclusion, this solution automates the traditionally manual task of BPMN diagram creation, offering significant improvements in speed, accuracy, and process optimization. By addressing key challenges such as redundancy detection and scalable diagram generation, it promises to revolutionize business process management in various industries

Research Objectives

Implementation of Text Extraction and Parsing System

This component uses Natural Language Processing (NLP) techniques to extract business process elements from unstructured text. The system processes text input, identifying key activities, tasks, and dependencies, and structures them into meaningful components. This step lays the foundation for generating accurate BPMN diagrams by transforming raw text into structured data.

Implementation of Redundancy Detection System

Leveraging semantic similarity algorithms, this component identifies redundant or duplicate tasks within the extracted business processes. By comparing the descriptions of tasks and analyzing their roles, the system detects overlaps and unnecessary steps, ensuring the process is optimized before moving to BPMN diagram generation.

Implementation of BPMN Diagram Generation

After processing the text and eliminating redundancies, this system component automatically generates BPMN diagrams. It maps the structured data to BPMN elements such as events, tasks, gateways, and lanes, ensuring the generated diagram accurately reflects the intended business process.

Implementation of Critical Path Analysis (CPA) Diagram Generation and Starvation Blocking Time Calculation

This component generates Critical Path Analysis (CPA) diagrams based on the processed data. It calculates the longest sequence of dependent tasks and identifies starvation and blocking times—delays caused by resource constraints—ensuring project timelines are accurate and efficient.

Methodology

Figure 2. High Level Architecture of the system.

The proposed Automated BPMN Diagram Generation System consists of 4 main components. They are

  1. Text Extraction and Parsing System
  2. Redundancy Detection and Optimization System
  3. BPMN Diagram Generation System
  4. Critical Path Analysis (CPA) and Starvation/Blocking Time Calculation

The operation of this comprehensive BPMN generation system is illustrated by the following process. The Text Extraction and Parsing System uses advanced NLP techniques to analyze unstructured text, extracting key process elements such as tasks and dependencies. The extracted data is then processed by the Redundancy Detection System, which compares the identified tasks, removing any redundant or duplicate steps. Once the data is optimized, the BPMN Diagram Generation System automatically generates the corresponding BPMN diagrams, mapping the structured data into visual components. Finally, the CPA and Starvation/Blocking Time Calculation System analyzes the diagram, identifying the critical path and calculating any potential delays due to resource bottlenecks. This system offers businesses a powerful tool for automating the creation of BPMN diagrams, improving process efficiency, and enhancing project management by identifying and resolving potential delays before they impact the project.
